Концепция

Философия Moon

Moon — это язык второго уровня для оркестрации искусственного интеллекта. Вместо того чтобы вшивать промпты в Python-скрипты, вы описываете агентов с типами, модели с ограничениями, и пайплайны в стиле Haskell (do, <-, pure).

🤖

Агенты как значения

Декларативные LLM-агенты с model, temperature, systemPrompt и произвольными полями конфигурации.

📐

Типы и схемы

Модели с полями, ограничениями (constraint:) и генерацией JSON-схем для валидации ответов LLM.

🔗

IO-пайплайны

Монадические do-блоки связывают вызовы API, память, файлы и агентский анализ в единый поток.

⛈️

Storm

Мульти-агентные панели обсуждают код, синтезатор объединяет вердикты — storm с panel и rounds.

Модели (model)

Модель описывает структуру данных с полями, опциональными ограничениями и источниками. Поддерживаются параметры типа (юкстапозиция, как в Haskell): model AnalysisResult t where.

moon
model AnalysisResult t where
  item       :: t
  findings   :: [Finding t]
  score      :: Float    constraint: between 0.0 100.0
  summary    :: String
  confidence :: Float    constraint: between 0.0 1.0

Модификаторы полей

МодификаторПример
constraint:constraint: between 0.0 1.0
default:default: "normal"
optionalПоле может отсутствовать в ответе
fetched fromВнешний источник данных

Типы данных (data)

moon
data Verdict = Approved | ChangesRequested | SecurityIssue

data Finding a = Finding
  { location    :: Location
  , severity    :: Severity
  , description :: String
  , category    :: Category a
  }

Агенты (agent)

Агент — конфигурация LLM с типовым классом (Analyzer t, Reviewer t и т.д.). Слово where опционально. Поля конфигурации произвольны.

moon
agent CodeAnalyzer :: Analyzer Code
  model: deepseek-v4-pro
  temperature: 0.18
  systemPrompt: """
    Ты эксперт по код-ревью с 15-летним опытом.
    Ищи проблемы архитектуры, безопасности и производительности.
  """

Вызов агента с контекстом (with)

moon
result <- CodeAnalyzer.analyze item
    with context: context
         maxTokens: 10000
         previousVersion: (getPreviousVersion item)
Блок with — layout-секция: пары ключ: значение на отдельных строках с отступом.

Функции и pattern matching

Сигнатура типа опциональна. Несколько уравнений — сопоставление с образцом.

moon
analyze :: Analyzer t -> t -> IO (AnalysisResult t)
analyze analyzer item = do
    context <- recall "project-knowledge"
    result  <- analyzer.analyze item
        with context: context
    pure $ AnalysisResult
        { item = item
        , findings = result.findings
        , score = calculateScore result
        }

    prs <- fetchOpenPRs "org/repo"
        filter: (not . isDraft)
    reviews <- mapM (analyze CodeAnalyzer) prs
    when (hasCriticalFindings reviews) $

Система типов

Типы записываются через -> (право-ассоциативно). Базовые типы — обычные идентификаторы из прелюдии, не зарезервированные слова.

moon
String  Int  Float  Bool
IO a                    -- монадические вычисления
Analyzer t               -- type class агента
[PullRequest]              -- список
Maybe Int                 -- применение типа
a -> b -> c               -- = a -> (b -> c)

Правило отступов (layout)

Как в Haskell: ключевые слова where, do, with открывают блок по колонке первого токена.

  • Та же колонка — новый элемент (неявная ;)
  • Больший отступ — продолжение предыдущего элемента
  • Меньший отступ — блок закрыт
Явные { } допустимы, но в реальном коде Moon используются только отступы.

Storm — мульти-агентные дебаты

Привязка storm запускает панель агентов, которые обсуждают артефакт (файл, путь), а synthesizer формирует итоговый консенсус.

moon
consensus <- storm "src/main.cpp"
    with panel: [SecurityReviewer, ArchitectReviewer]
         synthesizer: LeadSynthesizer
         rounds: 2

Параметры по умолчанию задаются в Moonfile: prompts.storm.default_rounds, prompts.storm.max_panel_size.

Moonfile

Манифест проекта Moonfile

Moonfile — декларативный манифест в стиле SwiftPM. Определяет зависимости stdlib, runnable targets, LLM-провайдеров, модели по умолчанию и runtime.

Файл называется Moonfile (без расширения). Расширение .moon зарезервировано для исходного кода.

Секции Moonfile

СекцияНазначение
package "name"Уникальное имя пакета (обязательно)
dependencies:Список модулей Core.*
targets:Именованные entrypoints → путь к .moon
providers:LLM-провайдеры (deepseek)
paths:Пути к pricing, tokenizer
models:default_flash, default_pro
prompts:Суффиксы, trace, настройки storm
runtime:Пул воркеров, память, ретраи

providers.deepseek

КлючОписание
api_keyenv("DEEPSEEK_API_KEY") — литералы sk-… запрещены
base_urlURL или env("DEEPSEEK_BASE_URL")
api_formatanthropic (по умолчанию) или openai
use_betaBeta OpenAI-совместимый API DeepSeek

runtime

КлючОписание
worker_pool.flash_concurrencyПараллельные flash-вызовы (по умолчанию 20)
worker_pool.pro_concurrencyПараллельные pro-вызовы (по умолчанию 5)

Секреты и env()

Moonfile намеренно не хранит секреты в открытом виде. Для ключей API используйте:

moonfile
providers:
  deepseek:
    api_key: env("DEEPSEEK_API_KEY")
    base_url: env("DEEPSEEK_BASE_URL")
    api_format: anthropic

Комментарии: -- и --? (документация для LSP hover в редакторе).

Полный пример Moonfile

moonfile
--? Moon project manifest (SwiftPM-style).
package "moon-lang-examples"

dependencies:
  Core.GitHub
  Core.Memory
  Core.Tools
  Core.FS
  Core.Network
  Core.Analyzers

  analyzer: examples/code-analyzer.moon
  reviewer: examples/code-reviewer.moon
  storm: examples/code-review-storm.moon

providers:
  deepseek:
    api_key: env("DEEPSEEK_API_KEY")
    base_url: env("DEEPSEEK_BASE_URL")
    api_format: anthropic

paths:
  pricing: docs/model-pricing.json
  tokenizer: deepseek-tokenizer/

models:
  default_flash: "deepseek-v4-flash"
  default_pro: "deepseek-v4-pro"

prompts:
  default_system_suffix: "Always respond with valid JSON matching the schema."
  trace_by_default: false
  storm:
    default_rounds: 1
    max_panel_size: 5

runtime:
  worker_pool:
    flash_concurrency: 20
    pro_concurrency: 5
  memory:
    long_term_backend: "file://.moon/memory"
  retries:
    max_repair_attempts: 1
